# 非構造化データ入口

このフォルダには、仕様化の入口となる Office 系ファイル（pptx, docx, xlsx）をカテゴリ別に配置します。

## 目的

- 既存のドキュメントを構造化されたマークダウン仕様に変換するための入力元
- 変換前の原本を保管し、トレーサビリティを確保する
- カテゴリ別管理により、どの業務領域の原本かを明確にする

## フォルダ構成

```
source/
├── 01_監視_モニタリング/
│   ├── pptx/     # PowerPoint ファイル
│   ├── docx/     # Word ファイル
│   ├── xlsx/     # Excel ファイル
│   └── pdf/      # PDF ファイル
├── 02_運用補佐ツール開発_管理/
│   ├── pptx/
│   ├── docx/
│   ├── xlsx/
│   └── pdf/
├── ...（03〜11 同構成）
└── 12_統制管理/
    ├── pptx/
    ├── docx/
    ├── xlsx/
    └── pdf/
```

## 一時置き場（staging）

`docs/staging/` は、非構造化データ → 構造化データ変換時の中間生成物を一時的に保管する場所です。

- AI が抽出した画像・テキスト・JSON などの中間ファイルを配置
- 変換完了後、成果物を該当カテゴリの SDD ステージフォルダへ移動
- staging 内のファイルは一時的なもので、Git にはコミットしない（`.gitignore` で除外）

## 運用ルール

1. Office 系ファイルを該当カテゴリのサブフォルダに配置する
2. AI（Claude Opus 4.6）を使って `docs/staging/` に中間データを抽出する
3. 中間データを構造化マークダウンに変換する
4. 変換後のマークダウンを `categories/` 配下の該当カテゴリ・SDD ステージフォルダに配置する
5. 変換元ファイルは `source/` にそのまま保管する（トレーサビリティ用）
6. staging 内の中間ファイルは変換完了後に適宜クリーンアップする
