# カテゴリ 12: 統制管理

## 概要
このカテゴリでは、インフラ業務の統制・コンプライアンス管理を目的とした、以下の成果物を対象とします：

- **監査基準定義**（ITIL / COBIT / 業界基準）
- **統制マトリックス**（予防/検出/是正）
- **統制証跡取得**（ログ、レポート、チェックリスト）
- **コンプライアンス対応**（規制監査対応）
- **監査ツール導入**（自動化）

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対象統制、監査基準など）
2. **Plan**: How を定義（マトリックス、ツール選定）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: 統制設計実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のマトリックス設計開始  
❌ requirements.md なしでの統制範囲定義  
❌ 品質ゲート不合格のまま次工程へ進む
