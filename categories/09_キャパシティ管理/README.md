# カテゴリ 09: キャパシティ管理

## 概要
このカテゴリでは、インフラ業務のキャパシティ・性能管理を目的とした、以下の成果物を対象とします：

- **キャパシティプラン**（1年/3年単位）
- **性能予測モデル**（トレンド分析、負荷予測）
- **リソースサイジング**（追加基準、安全マージン）
- **自動スケーリング設定**（トリガー、ルール）
- **ボトルネック分析・チューニング**

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（監視対象、予測精度など）
2. **Plan**: How を定義（予測モデル、ツール選定）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: キャパシティ設計実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後の予測モデル開発開始  
❌ requirements.md なしでの監視項目定義  
❌ 品質ゲート不合格のまま次工程へ進む
