# カテゴリ 01: 監視（モニタリング）

## 概要
このカテゴリでは、インフラ業務の監視・モニタリングを目的とした、以下の成果物を対象とします：

- **監視ダッシュボード**（Grafana、Azure Monitor、CloudWatch など）
- **アラート設定**（しきい値、通知ルール、エスカレーション）
- **ログ分析・監視**（ELK Stack、Splunk、Azure Log Analytics など）
- **メトリクス監視**（CPU、メモリ、ディスク、ネットワーク）
- **監視ポリシー・基準**（監視対象、精度、検知時間）

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（監視対象、なぜ監視するか）
2. **Plan**: How を定義（監視ツール、API設計、ダッシュボード構成）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: 監視設定・ダッシュボード実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後の直接実装  
❌ requirements.md なしでのダッシュボード設計  
❌ Specify段階での技術選定  
❌ 品質ゲート不合格のまま次工程へ進む
