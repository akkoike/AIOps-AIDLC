# カテゴリ 06: 構成管理・資産管理

## 概要
このカテゴリでは、インフラ業務の構成管理・資産管理を目的とした、以下の成果物を対象とします：

- **CMDB（Configuration Management Database）**
- **資産台帳**（VM、ストレージ、ライセンスなど）
- **構成差分監視**（Drift Detection）
- **CI（Configuration Item）管理**
- **構成バージョン管理**

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対象資産、精度目標など）
2. **Plan**: How を定義（スキーマ、ツール、収集方法）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: 構成設計実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のスキーマ設計開始  
❌ requirements.md なしでのCMDB属性定義  
❌ 品質ゲート不合格のまま次工程へ進む
