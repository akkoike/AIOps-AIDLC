# カテゴリ 07: セキュリティ管理

## 概要
このカテゴリでは、インフラ業務のセキュリティ管理を目的とした、以下の成果物を対象とします：

- **脆弱性スキャン**（頻度、対象、優先度）
- **パッチ管理**（適用ポリシー、テスト手順）
- **セキュリティ監査**（内部/外部、基準）
- **アクセス制御見直し**（権限監査、不正アクセス対応）
- **NIST / Zero-Trust対応**

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対象脆弱性、検知精度など）
2. **Plan**: How を定義（スキャンツール、対応フロー）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: セキュリティ設計実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物
