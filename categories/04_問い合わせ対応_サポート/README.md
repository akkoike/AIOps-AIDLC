# カテゴリ 04: 問い合わせ対応・サポート

## 概要
このカテゴリでは、インフラ業務のサポート・ナレッジ管理を目的とした、以下の成果物を対象とします：

- **FAQ（よくある質問）**
- **回答テンプレート**（形式、内容構成）
- **対応フロー**（L1/L2/L3 エスカレーション）
- **ナレッジベース整備**（Confluence、GitHub Wiki など）
- **サポート品質管理**（初回応答時間、解決時間）

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対応対象、カバー率目標など）
2. **Plan**: How を定義（FAQカテゴリ、テンプレート形式）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: FAQ・テンプレート実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のFAQ作成開始  
❌ requirements.md なしでのテンプレート定義  
❌ 品質ゲート不合格のまま次工程へ進む
