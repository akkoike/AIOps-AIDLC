# カテゴリ 05: 変更・リソース管理

## 概要
このカテゴリでは、インフラ業務の変更・リリース管理を目的とした、以下の成果物を対象とします：

- **変更申請プロセス**（Emergency / Standard / Minor）
- **CAB（変更諮問委員会）基準**（判定基準、メンバー構成）
- **リリース計画**（スケジュール、ロールバック手順）
- **変更パイプライン**（テスト → 本番適用 → 監視）
- **リリースカレンダー・体制**（実施時間、担当者）

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対象変更、成功率目標など）
2. **Plan**: How を定義（プロセス、ツール、基準）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: 変更プロセス実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のプロセス設計開始  
❌ requirements.md なしでのCAB基準定義  
❌ 品質ゲート不合格のまま次工程へ進む
