# カテゴリ 03: インシデント・障害対応 - 仕様優先実行モード

## 概要
このカテゴリで扱うすべての依頼（障害対応フロー、RCA手順、エスカレーション基準、初期対応SOP など）について、**仕様駆動開発（SDD）の7工程を厳密に守る**ことを目的とします。

## 強制される実行順序

```
1️⃣  ヒアリング（8項目）
        ↓
2️⃣  カテゴリ判定: 03_インシデント_障害対応 と確定
        ↓
3️⃣  01_specify/<request-folder>/requirements.md 作成
     （What/Why のみ。How は一切含めない）
        ↓
4️⃣  仕様品質ゲート実行
     スキル: sdd-requirements-quality-gate
     判定: 合格 を確認してから次へ
        ↓
5️⃣  02_plan/<request-folder>/plan.md 作成
     （技術スタック、実装方針を記述）
        ↓
6️⃣  03_tasks/<request-folder>/tasks.md 作成
     （30分以内の粒度でタスク分解）
        ↓
7️⃣  04_implement/<request-folder>/implement.md + 設計実装
     （ここで初めてSOP、決定木、RCAテンプレート実装を開始）
        ↓
8️⃣  05_verify/<request-folder>/verification.md 作成
     （テスト設計・検証実行）
        ↓
9️⃣  06_migration/<request-folder>/migration.md 作成
     （本番展開手順）
        ↓
🔟 output/<request-folder>/result.md 作成
     （最終成果物）
```

## SOP/手順書生成時の制約

### SOP生成を開始できる条件
以下をすべて満たすまで、SOP・決定木・RCAテンプレートなどの設計実装を開始しない：

- [ ] `01_specify/<request-folder>/requirements.md` が存在し、What/Why が完成している
- [ ] 仕様品質ゲート（`sdd-requirements-quality-gate`）で「合格」判定を得ている
- [ ] `02_plan/<request-folder>/plan.md` が完成し、対応フロー・ツール選定が確定している
- [ ] `03_tasks/<request-folder>/tasks.md` が完成し、タスク分解が確認されている
- [ ] ユーザーが明示的に「実装開始」を指示している

### SOP・ドキュメント配置ルール
```
✅ 正しい配置:
   categories/03_インシデント_障害対応/04_implement/<request-folder>/

❌ 間違った配置:
   categories/03_インシデント_障害対応/01_specify/<request-folder>/  ← 絶対禁止
   categories/03_インシデント_障害対応/02_plan/<request-folder>/     ← 絶対禁止
```

## 禁止事項（必読）

### ❌ 実装ファーストアプローチ
```
❌ 間違ったパターン:
   1. ヒアリングを受ける
   2. 直接SOP・決定木を設計
   3. 後付けで requirements.md を作成

✅ 正しいパターン:
   1. ヒアリングを受ける
   2. requirements.md（What/Why）を仕様品質ゲート合格まで作成
   3. plan.md（対応フロー、ツール選定）を作成
   4. tasks.md（タスク分解）を作成
   5. それから初めてSOP・RCAテンプレート設計
```

**最終原則**: 「仕様なき実装は許さず。常に仕様駆動で。」
