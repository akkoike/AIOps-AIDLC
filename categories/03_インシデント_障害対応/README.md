# カテゴリ 03: インシデント・障害対応

## 概要
このカテゴリでは、インフラ業務の障害対応を目的とした、以下の成果物を対象とします：

- **障害対応フロー**（初動対応、切り分け、エスカレーション）
- **RCA手順**（根本原因分析プロセス、テンプレート）
- **SOP（標準運用手順）**（障害検知〜復旧）
- **意思決定木**（トラブルシューティング）
- **エスカレーション基準**（条件、対象者）

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対応対象、MTTR目標など）
2. **Plan**: How を定義（フロー、ツール、手順）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: SOP・決定木・RCAテンプレート実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のSOP設計開始  
❌ requirements.md なしでのプロセス定義  
❌ Specify段階での実装詳細検討  
❌ 品質ゲート不合格のまま次工程へ進む
