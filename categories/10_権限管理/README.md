# カテゴリ 10: 権限管理

## 概要
このカテゴリでは、インフラ業務のアクセス制御・権限管理を目的とした、以下の成果物を対象とします：

- **IAM（Identity & Access Management）体系**
- **ロール定義**（管理者、開発者、運用チーム など）
- **プロビジョニング自動化**（申請 → 承認 → 付与）
- **アクセス棚卸し**（定期レビュー）
- **認証情報管理**（MFA、シークレット管理）

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（対象ロール、棚卸し頻度など）
2. **Plan**: How を定義（IAMツール、ロール体系）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: IAM設計実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のIAMツール導入開始  
❌ requirements.md なしでのロール定義  
❌ 品質ゲート不合格のまま次工程へ進む
