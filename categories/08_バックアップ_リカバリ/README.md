# カテゴリ 08: バックアップ・リカバリ

## 概要
このカテゴリでは、インフラ業務のバックアップ・災害復旧を目的とした、以下の成果物を対象とします：

- **バックアップ戦略**（フル/増分/差分、保持ポリシー）
- **RPO/RTO設定**（目標値、実績管理）
- **DR（災害復旧）計画**（フェイルオーバー手順、優先度）
- **復旧テスト**（頻度、手順、検証項目）
- **BCDR（事業継続）対応**

## SDD 7工程の実行順序（必須）

このカテゴリで取り扱うすべての依頼は、以下の工程順序を **厳密に守る** 必要があります：

1. **Specify**: What/Why を定義（バックアップ対象、RPO目標など）
2. **Plan**: How を定義（ストレージ戦略、ツール選定）
3. **Tasks**: タスク分解（粒度: 30分以内）
4. **Implement**: バックアップ設計実装
5. **Verify**: テスト・検証実行
6. **Migration**: 本番展開手順
7. **Output**: 最終成果物

## 禁止事項

❌ ヒアリング直後のツール導入開始  
❌ requirements.md なしでのバックアップスケジュール定義  
❌ 品質ゲート不合格のまま次工程へ進む
